Passo a passo para criar uma aplicação de consulta de CEP:

Definir a URL base da API:

Utilizar a classe ApiConstants para definir a URL base da API, que no caso é "https://viacep.com.br/ws/$cep/json/".
Definir o caminho para os ativos (assets):

Utilizar a classe PathAssets para definir o caminho para os ativos (assets), como por exemplo o ícone de localização.
Criar a classe de modelo de endereço (AddressModel):

Definir os atributos do modelo de endereço, como cep, logradouro, complemento, bairro, localidade, uf, ibge, ddd, etc.
Implementar métodos para converter de/para JSON.
Criar o widget para exibir o endereço (CardLocationWidget):

Implementar um widget que recebe um modelo de endereço como parâmetro e exibe suas informações em um cartão.
Criar a classe de controle da tela inicial (HomeController):

Utilizar a classe ChangeNotifier para gerenciar o estado da tela.
Implementar métodos para buscar um endereço com base no CEP e limpar o estado.
Criar a tela inicial (HomeScreen):

Implementar a interface da tela inicial, incluindo um campo de texto para inserir o CEP, botão para consultar, indicador de carregamento, e exibição do endereço quando disponível.
Implementar o serviço de API (ApiService):

Utilizar a biblioteca Dio ou HTTP para fazer solicitações HTTP à API ViaCEP.
Implementar um método para buscar um endereço com base no CEP fornecido.
Configurar a aplicação:

Definir a configuração inicial da aplicação, como tema, título e tela inicial.
Utilizar a classe MaterialApp para configurar a aplicação Flutter.
Conversão do modelo JSON para Dart:

Utilizar um conversor online para converter o modelo JSON de endereço em uma classe Dart.
Criar uma classe Dart com os atributos correspondentes ao modelo JSON.
Arquivo de constantes da API (api_constants):

Criar um arquivo para armazenar constantes relacionadas à API, como a URL base.
